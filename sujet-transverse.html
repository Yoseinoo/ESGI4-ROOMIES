Sujet du Projet "Roomies"

Objectif :

Créer un site de jeux en ligne permettant aux utilisateurs de discuter et de jouer en temps réel.

Fonctionnalités Obligatoires :

  1. Page d'Accueil (HomePage) :

    • Chat global accessible sur tout le site.
    • Liste des "rooms" avec indication du nombre de joueurs présents (ex. 0/2, 1/2).
    • Filtres pour sélectionner une room spécifique.

  2. Page des Rooms (RoomPage) :

    • Affichage du jeu en cours dans la room.
    • Chat spécifique à chaque room.

  3. Gestion des Utilisateurs :

    • Système de login/logout.
    • CRUD (Créer, Lire, Mettre à jour, Supprimer) pour les utilisateurs.
    • Page de profil utilisateur.

  4. Gestion des Rooms :

    • Création et suppression de rooms.
    • Gestion du nombre de joueurs dans une room.

  5. Fonctionnalités de Base des Jeux :

    • Implémentation d'au moins un jeu (ex. morpion).

  6. Communication en Temps Réel :

    • Utilisation de WebSocket pour le chat en temps réel. D'accord, nous allons réviser les fonctionnalités optionnelles pour qu'elles soient transverses et puissent être évaluées dans chaque grille, que ce soit pour Vue.js, Symfony, les tests d'application, ou la communication en temps réel. Fonctionnalités Optionnelles Transverses

Deux fonctionnalités sont à choisir et implémenter parmi les 6 suivantes :

  1. Jeux supplémentaires ou personnalisation :

    • Ajout de nouveaux jeux (ex. memory, puzzle) ou options de personnalisation des jeux existants.
    • Évaluation : Implémentation frontend (Vue.js), logique backend (Symfony), tests associés, et communication en temps réel si applicable.

  2. Améliorations du chat :

    • Messages privés entre utilisateurs, notifications pour les messages non lus, ou historique de chat.
    • Évaluation : Interface utilisateur (Vue.js), gestion backend (Symfony), tests des fonctionnalités, et intégration WebSocket.

  3. Interface utilisateur avancée :

    • Thèmes personnalisables pour l'interface, accessibilité améliorée (support des lecteurs d'écran), ou améliorations ergonomiques.
    • Évaluation : Design et composants frontend (Vue.js), support backend pour les thèmes (Symfony), tests d'accessibilité, et performance de l'interface.

  4. Intégration avec des API externes :

    • Intégration avec une API de discussion en temps réel ou utilisation d'API pour des fonctionnalités supplémentaires (ex. géolocalisation).
    • Évaluation : Intégration frontend et backend, tests des intégrations, et communication en temps réel.

  5. Fonctionnalités sociales :

    • Système d'amis ou de suivi d'utilisateurs, tableau de classement pour les jeux, ou interactions sociales dans le chat.
    • Évaluation : Composants frontend (Vue.js), logique backend (Symfony), tests des interactions, et notifications en temps réel.

  6. WebRTC pour la communication en temps réel :

    • Implémentation de WebRTC pour des fonctionnalités avancées comme le partage d'écran ou les appels vidéo.
    • Évaluation : Intégration frontend et backend, tests des communications, et performance de WebRTC.
